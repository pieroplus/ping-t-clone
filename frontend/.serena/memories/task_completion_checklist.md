# タスク完了時のチェックリスト

コードの変更を行った後は、以下のチェックを実施してください。

## 1. リントチェック
```bash
npm run lint
```
- ESLintエラーがないことを確認
- 警告も可能な限り解消

## 2. TypeScriptエラーチェック
```bash
npm run build
```
- TypeScriptの型エラーがないことを確認
- ビルドが成功することを確認

## 3. 動作確認
```bash
npm run dev
```
- 開発サーバーを起動
- 変更した機能が正しく動作することをブラウザで確認
- エラーがコンソールに出ていないか確認

## 4. コードレビュー（セルフチェック）
- [ ] 命名規則に従っているか
- [ ] 適切な型定義がされているか
- [ ] 不要なコメントやコンソールログが残っていないか
- [ ] インポートが整理されているか
- [ ] 未使用の変数やインポートがないか

## 5. Git コミット
変更が適切であれば、変更をコミット：
```bash
git add .
git commit -m "meaningful commit message"
```

## Phase A の機能別チェック

### 認証機能の変更時
- [ ] ログイン/ログアウトが正しく動作するか
- [ ] トークンがlocalStorageに保存されるか
- [ ] 認証エラー時に適切にリダイレクトされるか

### タイトル一覧の変更時
- [ ] タイトルが正しく表示されるか
- [ ] ページネーションが動作するか
- [ ] APIエラー時の処理が適切か

### 出題モードの変更時
- [ ] 問題が正しく表示されるか
- [ ] 選択肢の選択が正しく動作するか（単一/複数）
- [ ] 採点が正しく行われるか
- [ ] 解説が表示されるか
- [ ] Toastメッセージが適切に表示されるか

## 推奨事項
- 大きな変更を行う前にブランチを作成
- 小さな単位で頻繁にコミット
- コミットメッセージは明確で意味のあるものに
